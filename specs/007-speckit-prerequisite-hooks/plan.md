# Implementation Plan: Speckit Prerequisite Guardrail Hooks

**Branch**: `007-speckit-prerequisite-hooks` | **Date**: 2026-07-07 | **Spec**: [spec.md](./spec.md)

**Status**: Approved — ready for `/speckit-tasks`

## Summary

Add mandatory prerequisite guardrail hooks to the Spec-Kit workflow by creating `.specify/extensions.yml`
(with `before_tasks` and `before_implement` hook registrations), two lightweight hook skills
(`speckit-check-prerequisites` and `speckit-check-prerequisites-implement`), and extending
`check-prerequisites.sh` with a `--require-spec` flag. When a developer runs `/speckit-tasks` or
`/speckit-implement` without the required upstream artifacts (`spec.md`, `plan.md`, `tasks.md`), the
mandatory hook fires first, outputs a clear error naming the missing file and the corrective command,
and halts the skill before any generation or implementation begins.

## Implementation Details

### 1. `.specify/extensions.yml` (new file)

```yaml
hooks:
  before_tasks:
    - extension: "Prerequisite Guardrail"
      command: "speckit.check-prerequisites"
      description: "Verify spec.md and plan.md exist before generating tasks"
      optional: false
      enabled: true

  before_implement:
    - extension: "Prerequisite Guardrail"
      command: "speckit.check-prerequisites-implement"
      description: "Verify spec.md, plan.md, and tasks.md exist before implementing"
      optional: false
      enabled: true
```

### 2. `check-prerequisites.sh` changes

Add `--require-spec` flag parsing alongside existing flags:

```bash
REQUIRE_SPEC=false
# In arg-parsing loop:
--require-spec) REQUIRE_SPEC=true ;;
```

Add spec.md check (after feature directory check, before plan.md check):

```bash
if $REQUIRE_SPEC && [[ ! -f "$FEATURE_SPEC" ]]; then
    echo "ERROR: spec.md not found in $FEATURE_DIR" >&2
    echo "Run /speckit-specify first to create the feature specification." >&2
    exit 1
fi
```

No changes to existing flag behavior or output format.

### 3. `.claude/skills/speckit-check-prerequisites/SKILL.md` (new file)

Hook skill invoked before `/speckit-tasks`. Runs `check-prerequisites.sh --require-spec` (checks spec.md + plan.md). Reports pass or halt.

Key skill logic:
- Resolve the current feature directory (from `.specify/feature.json` via SPECIFY_FEATURE_DIRECTORY workaround)
- Run `check-prerequisites.sh --require-spec` via bash
- If exit code is non-zero OR stderr contains ERROR: → output HALT message with the error details
- If exit code is 0 → output pass confirmation

### 4. `.claude/skills/speckit-check-prerequisites-implement/SKILL.md` (new file)

Hook skill invoked before `/speckit-implement`. Runs `check-prerequisites.sh --require-spec --require-tasks` (checks spec.md + plan.md + tasks.md).

Same structure as above but with `--require-tasks` added.
